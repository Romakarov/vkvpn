# Work on Task from PROJECT_PLAN

## Алгоритм

### 1. Прочитать план
Прочитай `PROJECT_PLAN.md` целиком. Найди:
- Если указан номер задачи — взять её
- Если "next" или не указано — взять задачу с наивысшим приоритетом, у которой:
  - Нет незавершённых зависимостей
  - Размер [S] или [M] (для [L] — спросить пользователя)

### 2. Подготовка
```bash
git checkout main && git pull origin main
git checkout -b claude/<task-name>
```

### 3. Выполнение
- Прочитать все файлы, указанные в задаче
- Реализовать по критериям "Готово когда"
- Коммитить после каждого логического шага
- Прогнать тесты: `go test ./server/ ./pkg/... -v -race`

### 4. Финализация
```bash
# Финальные тесты
go vet ./...
go test ./server/ ./pkg/... -v -race

# Push
git push -u origin claude/<task-name>
```

### 5. Создать PR
Создать PR в `main` с описанием:
- Какая задача из плана
- Что сделано
- Что протестировано

### 6. Доложить
Сообщить пользователю:
- Задача X.Y выполнена
- PR создан: <ссылка>
- Тесты: passed/failed
- Что следующее по плану

## Правила
- ОДНА задача за раз, не хвататься за всё
- Коммит после каждого логического шага, НЕ копить
- Тесты перед каждым коммитом
- Если задача блокирована — доложить и предложить альтернативу
- Если задача требует SSH/VPS — спросить доступы у пользователя
- НЕ трогать `releases/`
- НЕ ломать обратную совместимость API
